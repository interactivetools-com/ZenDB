# Multiple Connections

For most applications, the `DB::connect()` singleton is all you need. If you
need to work with multiple databases or different settings, there are two
approaches.

## Separate Connections

Create `Connection` instances directly to connect to different databases:

```php
use Itools\ZenDB\Connection;

$analytics = new Connection([
    'hostname' => 'analytics-db.example.com',
    'username' => 'reader',
    'password' => 'secret',
    'database' => 'analytics',
]);

$rows = $analytics->select('events', "created_at > NOW() - INTERVAL ? DAY", 1);
```

Each `Connection` instance has the same query methods as `DB::` (`select`,
`selectOne`, `insert`, `update`, `delete`, `count`, `query`, `queryOne`).

## Cloning a Connection

`DB::clone()` creates a new Connection that shares the same underlying mysqli
connection but with different settings. The original connection is unaffected.

```php
$db = DB::clone(['useSmartJoins' => false]);
$rows = $db->query("SELECT u.name, o.total_amount
                     FROM ::users u JOIN ::orders o ON u.id = o.user_id");
```

Clones are useful when you need different query behavior (like disabling Smart
Joins) without opening a second database connection.

You cannot override credentials (`hostname`, `username`, `password`, `database`)
in a clone since it shares the source connection.
